# milk-pm

A configurable web dashboard for [Remember The Milk](https://www.rememberthemilk.com). Surfaces your RTM lists as a lightweight project management dashboard — grouped by project, with task views, stats, and more.

Works out of the box with [milk-mcp](https://github.com/donkitchen/milk-mcp)'s list conventions. Bring your own RTM list structure via `projects.json`.

---

## What it looks like

- **Dashboard** — all your projects at a glance, with open task counts and bug badges
- **Project detail** — drill into any project to see TODO, Backlog, Bugs, Decisions, and Context lists
- **Stats bar** — total open items and bugs across all projects

---

## Prerequisites

1. A [Remember The Milk](https://www.rememberthemilk.com) account (free tier works)
2. RTM API credentials — get them at [rememberthemilk.com/services/api/keys.rtm](https://www.rememberthemilk.com/services/api/keys.rtm)
3. Node.js 18+

## Setup

### 1. Clone and install

```bash
git clone https://github.com/donkitchen/milk-pm.git
cd milk-pm
npm install
```

### 2. Configure environment

```bash
cp .env.example .env.local
```

Edit `.env.local` and add your RTM API key and shared secret:

```
RTM_API_KEY=your_api_key_here
RTM_SHARED_SECRET=your_shared_secret_here
```

### 3. Authenticate with RTM

Run the auth script to get your token:

```bash
npm run auth
```

This will open a browser URL for you to authorize the app, then write your `RTM_AUTH_TOKEN` to `.env.local`.

> **Already using milk-mcp?** Your credentials are in `~/.config/milk-mcp/config`. Copy the three values into `.env.local` and skip the auth step.

### 4. Configure your projects

Edit `projects.json` to map your RTM lists to projects:

```json
{
  "projects": [
    {
      "slug": "my-project",
      "name": "My Project",
      "description": "Optional description",
      "color": "blue",
      "url": "https://github.com/you/my-project",
      "lists": {
        "todo":      "CC: my-project - TODO",
        "backlog":   "CC: my-project - Backlog",
        "bugs":      "CC: my-project - Bugs",
        "decisions": "CC: my-project - Decisions",
        "context":   "CC: my-project - Context"
      }
    }
  ]
}
```

The `lists` values are the **exact names** of your RTM lists. You can use any naming convention — milk-pm just needs to know which list plays which role.

**Available colors:** `blue` `teal` `purple` `amber` `green` `red` `gray`

### 5. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Deploying to Vercel

```bash
npx vercel
```

Add the following environment variables in your Vercel project settings:

| Variable | Value |
|---|---|
| `RTM_API_KEY` | Your RTM API key |
| `RTM_SHARED_SECRET` | Your RTM shared secret |
| `RTM_AUTH_TOKEN` | The token generated by `npm run auth` |

## Using with milk-mcp

[milk-mcp](https://github.com/donkitchen/milk-mcp) is an MCP server that gives Claude Code persistent memory through RTM. It creates five lists per project using the naming convention:

```
CC: [project-name] - TODO
CC: [project-name] - Backlog
CC: [project-name] - Bugs
CC: [project-name] - Decisions
CC: [project-name] - Context
```

milk-pm's default `projects.json` example follows this convention exactly. If you're already using milk-mcp, your RTM credentials will work as-is — just copy them from `~/.config/milk-mcp/config`.

---

## Bring your own convention

milk-pm doesn't care what your RTM lists are called. The `lists` object in `projects.json` maps each role to an exact RTM list name, so any naming scheme works:

```json
"lists": {
  "todo":      "proj:myapp:todo",
  "backlog":   "proj:myapp:backlog",
  "bugs":      "proj:myapp:bugs",
  "decisions": "proj:myapp:decisions",
  "context":   "proj:myapp:context"
}
```
